# 检查点模板

本文档定义各类检查点的标准格式。

## 一、阶段检查点模板

```markdown

## 基本信息

| 项目 | 内容 |
|------|------|
| 检查点ID | cp-stage-XXX |
| 时间 | YYYY-MM-DD HH:MM:SS |
| 阶段 | [阶段名称] |
| 状态 | 完成/进行中 |

## 阶段产出

### 已完成文件

| 文件 | 路径 | 状态 |
|------|------|------|
| [文件名] | [路径] | ✓ 完成 |

### 阶段决策

| 决策ID | 主题 | 结果 |
|--------|------|------|
| d001 | [主题] | [结果] |

## 进度统计

| 指标 | 数值 |
|------|------|
| 本阶段Token | 0 |
| 本阶段成本 | ¥0.00 |
| 累计Token | 0 |
| 累计成本 | ¥0.00 |

## 下一阶段

- 阶段：[下一阶段名称]
- 首个动作：[动作描述]
- 负责角色：[角色]

---
**保存人**：[角色]
```

## 二、批次检查点模板

```markdown
# 批次检查点 - 第X-Y章

## 基本信息

| 项目 | 内容 |
|------|------|
| 检查点ID | cp-batch-XXX |

| 状态 | 完成/进行中 |

## 章节进度

| 章节 | 状态 | 爽点 | 字数 | 审核结果 |
|------|------|------|------|----------|
| X | ✓ 完成 | [爽点] | 2500 | 通过 |
| X+1 | ✓ 完成 | [爽点] | 2500 | 通过 |
| X+2 | ✓ 完成 | [爽点] | 2500 | 通过 |
| X+3 | ✓ 完成 | [爽点] | 2500 | 通过 |
| X+4 | ✓ 完成 | [爽点] | 2500 | 通过 |

## 已生成文件

```
outputs/

...

### 人物状态

| 角色 | 当前状态 | 实力变化 |

| 主角 | [状态] | [变化] |

### 伏笔追踪

| 伏笔 | 埋入章节 | 计划回收章节 | 状态 |
|------|----------|--------------|------|
| [伏笔] | 第X章 | 第Y章 | 待回收 |

## 下一批次

- 批次：第X+5 - 第Y+5章

**保存人**：[角色]

## 六、检查点命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 阶段 | cp-stage-XXX | cp-stage-001 |
| 批次 | cp-batch-XXX | cp-batch-001 |
| 章节 | cp-chapter-XXX | cp-chapter-001 |
| 决策 | cp-decision-XXX | cp-decision-001 |
| 紧急 | cp-emergency-XXX | cp-emergency-001 |

**编号规则**：三位数字，按顺序递增
