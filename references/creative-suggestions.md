# 研究员创意建议框架

本文档定义研究员在调研完成后可以提供的创意建议类型和格式。

---

## 一、建议类型

### 1.1 书名建议

研究员需提供多个书名选项：

```markdown
## 书名建议

| 选项 | 书名 | 亮点 | 建议 |
|------|------|------|------|
| 1 | [书名A] | [亮点] | ⭐推荐 |
| 2 | [书名B] | [亮点] | |
| 3 | [书名C] | [亮点] | |
```

**书名原则**：
- 体现核心卖点
- 简洁有吸引力
- 符合平台风格
- 便于搜索

### 1.2 核心卖点建议

```markdown
## 核心卖点建议

### 主打卖点（必选）
- 卖点1：[描述]

### 辅助卖点（可选）
- 卖点2：[描述]
- 卖点3：[描述]

### 差异化亮点
- [与同类型作品的差异点]
```

## 三、建议原则

### 3.1 多样性原则

每个类型至少提供2-3个选项，让总编和用户有选择空间。

### 3.2 数据支撑

建议需要有数据/案例支撑，不能凭空想象。

### 3.3 差异化原则

建议要体现与现有作品的差异化，避免同质化。

### 3.4 可执行性

建议要具体可执行，不是空泛的概念。

---

## 四、使用流程

1. **研究员调研** → 产出研究简报
2. **研究员思考** → 产出创意建议
3. **编辑审核** → 检查建议合理性
4. **总编呈交** → 由用户选择确认
5. **写手执行** → 根据确认的建议撰写

---

## 五、与大纲的关系

- **创意建议**：方向性的、多选项的
- **大纲**：确定性的、单路径的

创意建议是为了帮助用户做决策，大纲是确认方向后的具体执行方案。
