# 编辑审核工作流

本文档包含编辑审核、AI去味等核心规则。

---

## 强制性门禁机制（监督与自动纠正⭐⭐⭐）

### 门禁核心原则

1. **记忆库是唯一真理**
   - 任何与记忆库不符的内容必须纠正
   - 写手不得私自修改设定

2. **AI痕迹零容忍**
   - 每次产出必须通过 AI 去味检查
   - 可疑处必须标记

3. **流程必须完整**
   - 写作前检查清单必须全部勾选
   - 章节完成后更新必须全部完成

### 自动纠正触发条件

| 情况 | 纠正动作 |
|------|----------|
| 设定与记忆库冲突 | 立即暂停，要求核对记忆库 |
| AI 痕迹明显 | 标记可疑处，返回写手修改 |
| 流程未完整 | 阻止进入下一阶段 |

## AI去味规则

### 五条核心规则

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 去除开场白、强调性拐杖词（"值得注意的是""需要强调的是"） |
| 打破公式结构 | 避免二元对比、三段式列举、修辞性设置 |
| 变化节奏 | 长短句混合，段落结尾多样化 |
| 信任读者 | 直接陈述事实，不过度解释 |
| 删除金句 | 不写可引用式的"名言警句" |

### 高频问题词汇（禁止使用）

- 此外、与此同时、至关重要、深入探讨、持久的、显著的、作为...代表
- **禁止结构**：不仅...而且...、从X到Y、这不仅仅是...而是...

### 产出后自检

每次提交前检查：
- [ ] 本章爽点是否明确？
- [ ] 是否有钩子吸引读者往下看？
- [ ] 主角是否有进展/收获？
- [ ] 有没有废话/水字数？
- [ ] 对话是否精炼？
- [ ] 断章是否恰当？
- [ ] **AI痕迹初步标记**（标记可疑处，提交给编辑判断）

---

## 审核标准

### 基础要求

- 每章 3000-5000 字
- 节奏明快，不拖沓
- 爽点清晰，情绪到位
- 断章有悬念
- **无AI写作痕迹**

### 进阶要求

- 场景画面感强
- 人物性格鲜明
- 斗法/冲突有张力
- 伏笔埋设自然

### 审核检查清单

| 检查项 | 要求 |
|--------|------|
| 剧情 | 与大纲一致 |
| 人物 | 性格稳定 |
| 节奏 | 爽点清晰 |
| 断章 | 有悬念 |
| AI痕迹 | 无明显问题 |
| 记忆库 | 已更新 |

## 编辑章节摘要整理（审核通过后必须执行⭐）

### 模板

```markdown
## 第X章 章节摘要

### 基本信息
- 章节名：[]
- 字数：[]
- 爽点：[核心爽点]

### 剧情摘要
[100-200字概括本章剧情]

### 人物变化
- 主角：[状态变化]
- 配角：[出场/变化]

### 伏笔
- 埋入：[]
- 回收：[]

### 待办
- [ ]
```

## 读者反馈（每12章）

### 触发条件

每完成 12 章写作后，启动读者反馈流程。

### 流程

1. **研究员阅读**
   - 阅读 1-12 章全文
   - 记录阅读感受

2. **生成反馈报告**
   - 优点分析
   - 问题点
   - 改进建议

3. **团队讨论**
   - 讨论反馈内容
   - 决定是否修改

### 反馈报告模板

```markdown
# 读者反馈报告 - 第X-Y章

## 整体感受
[ ]

## 优点
- [ ]

## 问题点
- [ ]

## 改进建议
- [ ]

## 团队决定
[ ] 需要修改 / [ ] 继续
```
