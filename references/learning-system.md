# 系统自动学习机制

本文档定义网文编辑部的自动学习和升级机制。

---

## 一、学习触发时机

### 1.1 项目级学习

每个项目完成后自动触发：
- 项目复盘
- 提取经验教训
- 更新方法论

### 1.2 批次级学习

每10章审核完成后触发：
- 节奏把控分析
- 爽点密度评估
- 问题模式识别

### 1.3 实时学习

审核过程中触发：
- 典型问题记录
- 修改建议库积累

---

## 二、学习内容

### 2.1 成功经验

- 哪些爽点设计效果好
- 哪些节奏安排合理
- 哪些人物受欢迎
- 哪些情节读者反馈好

### 2.2 失败教训

- 哪些写法效果差
- 哪些节奏有问题
- 哪些情节被读者吐槽
- 审核中反复出现的问题

### 2.3 优化建议

- 方法论需要调整的地方
- 清单需要补充的检查项
- 模板需要优化的格式

---

## 三、学习流程

### 3.1 自动记录

系统在工作中自动记录：

```
outputs/
├── learning/
│   ├── project-001/
│   │   ├── feedback.md         # 项目反馈
│   │   ├── issues.md          # 问题记录
│   │   └── improvements.md    # 改进建议
│   └── ...
```

### 3.2 定期复盘

总编定期进行复盘：

1. **收集反馈**：汇总所有学习记录
2. **识别模式**：找出反复出现的问题
3. **提取规律**：总结成功/失败模式
4. **生成建议**：提出具体的改进方案

### 3.3 知识沉淀

复盘结果沉淀到对应文件：

| 问题类型 | 写入文件 |
|----------|----------|
| 写作方法问题 | methodology.md |
| 审核标准问题 | checklist.md |
| 格式规范问题 | CONVENTIONS.md |
| 模板问题 | templates/ |
| 偏好问题 | USER.md |
| 新案例 | case-library/ |

---

## 四、自动升级机制

### 4.1 版本管理

方法论和清单采用版本管理：

```
methodology.md
├── v1.0.md   # 初始版本
├── v1.1.md   # 小幅优化
└── v2.0.md   # 重大升级
```

### 4.2 升级规则

- **小幅优化 (x.1)**：新增检查项、补充说明
- **重大升级 (x.0)**：结构调整、体系重写

### 4.3 升级流程

1. 总编生成升级建议
2. 向你确认是否执行
3. 执行后更新版本号
4. 记录升级原因

---

## 五、学习提示

### 5.1 研究员学习提示

研究员在研究时可以：
- 参考案例库中的成功案例
- 分析当前热门作品的套路
- 提取可复用的模式

### 5.2 写手学习提示

写手在撰写时可以：
- 参考同类型成功作品
- 运用案例库中的技巧
- 避免常见错误

### 5.3 编辑学习提示

编辑在审核时可以：
- 对照案例库判断
- 记录新的典型问题
- 补充检查清单

---

## 六、效果评估

### 6.1 量化指标

- 审核通过率
- 修改次数
- 读者反馈
- 平台数据

### 6.2 质性评估

- 爽点密度变化
- 节奏紧凑度
- 人物鲜活度

### 6.3 持续改进

每次评估后：
- 识别新问题
- 提出改进方案
- 写入方法论
