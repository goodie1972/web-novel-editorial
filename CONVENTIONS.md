# 项目规范与设计规则

## 文件命名规范

### 输入文件

```
source/
├── requirements.md           # 创作需求
└── materials/               # 参考素材
    ├── 01-参考作品.md
    └── 02-设定资料.md
```

### 输出文件

```
outputs/
├── research-brief.md        # 研究简报
├── outline.md               # 大纲
├── world/                   # 世界观设定
│   ├── 修炼体系.md
│   ├── 势力分布.md
│   └── 角色设定.md
├── chapters/                # 章节内容
│   ├── chapter-001.md
│   ├── chapter-002.md
│   └── ...
└── reviews/                 # 审核意见
    ├── review-outline.md
    └── review-chapters-01-10.md
```

## 章节命名规范

- 格式：`chapter-XXX.md`
- 序号三位数：001, 002, ... 099, 100
- 每10章一个审核批次

## 内容格式规范

### 章节模板

```markdown
# 第X章 章节标题

[正文内容]

---

**字数**：XXX字
**爽点**：XXX
**钩子**：XXX
```

### 大纲模板

```markdown
# [书名] 大纲

## 基本信息
- 题材：
- 风格：
- 预计字数：
- 目标平台：

## 核心设定

### 主角
- 姓名：
- 性格：
- 金手指：
- 目标：

### 世界观
[世界观描述]

## 爽点规划
[全文爽点布局]

## 结构设计
[章节规划]

## 伏笔设计
[伏笔列表]
```

## 审核记录规范

```markdown
# 审核意见：[内容类型]

## 审核结果：通过 / 不通过

### 问题列表

| 位置 | 问题 | 标准依据 | 修改建议 |
|------|------|----------|----------|
| 第X章第Y段 | 描述 | checklist.md第Z条 | 建议 |

### 通过项
- [ ] 检查项1
- [ ] 检查项2
```

## 版本管理

- 初稿：draft-v1
- 修改后：draft-v2, draft-v3
- 终稿：final

## 质量门槛

- 每章最低3000字
- 每章至少1个爽点
- 每章必须有钩子
- 黄金三章必须过审

## 交付标准

### 可商用标准

- 全文无敏感词
- 符合平台规范
- 达到签约字数
- 节奏符合网文标准
- 爽点密度足够
