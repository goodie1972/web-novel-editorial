# 网文写手技能包

## 技能概述

本技能包定义网文撰写环节的执行步骤和质量标准。

## 触发条件

- 收到研究简报和创作大纲
- 需要将大纲扩展为具体章节内容

## 记忆库使用规则（必须遵守⭐）

**全员讨论会后，所有资料必须通过记忆库获取。写作前必须查询记忆库。**

### 写作前查询（必须）

```
开始写作 → 从记忆库查询设定 → 基于查询结果写作 → 完成后将摘要写入记忆库
```

查询内容：
| 查询项 | 集合 | 用途 |
|--------|------|------|
| 人物设定 | characters | 确保人物性格、外貌、行为一致 |
| 世界观 | world | 确保设定细节准确 |
| 技能设定 | skills | 确保技能效果、威力一致 |
| 最近章节 | chapters | 确保情节衔接 |
| 伏笔 | foreshadowing | 确认本章需要埋入/回收的伏笔 |

详细配置见 [chroma-integration.md](../../references/chroma-integration.md)

## AI去味要求（必须遵守⭐）

**所有产出必须去除AI写作痕迹，这是写手铁律。** 详见 [humanizer.md](../../references/humanizer.md)

### 五条核心规则

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 去除开场白、强调性拐杖词（"值得注意的是""需要强调的是"） |
| 打破公式结构 | 避免二元对比、三段式列举、修辞性设置 |
| 变化节奏 | 长短句混合，段落结尾多样化 |
| 信任读者 | 直接陈述事实，不过度解释 |
| 删除金句 | 不写可引用式的"名言警句" |

### 高频问题词汇

**禁止使用**：此外、与此同时、至关重要、深入探讨、持久的、显著的、作为...代表

**禁止结构**：不仅...而且...、从X到Y、这不仅仅是...而是...

### 产出前必查

每次提交前必须自查：
- [ ] 是否有填充短语？
- [ ] 是否有三段式列举？
- [ ] 句子长度是否变化？
- [ ] 是否过度解释？
- [ ] 是否有"金句"式表达？

## 工作流程

### 第一步：确认大纲

收到编辑审核通过的大纲后，写手需：

1. 仔细阅读大纲，理解整体结构和爽点布局
2. 确认每一章的核心爽点
3. 确认人物设定和世界观细节
4. 如有疑问，先与编辑沟通

### 第二步：世界观构建

根据大纲需求，完善世界观设定：

1. 修炼体系（如果需要）
2. 势力分布
3. 货币/资源体系
4. 重要地名/场所

### 第三步：章节撰写

按照以下模板结构撰写每一章：

**章开篇**：
- 承接上文钩子（200-300字）
- 本章背景引入

**章中段**：
- 情节铺垫（500-1000字）
- 冲突种子埋入

**章高潮**：
- 核心冲突爆发（1000-2000字）
- 爽点释放

**章结尾**：
- 胜负已分或悬念留下
- 甩出下一章钩子

### 第四步：自检

完成每章后，自检以下内容：

- [ ] 本章爽点是否明确？
- [ ] 是否有钩子吸引读者往下看？
- [ ] 主角是否有进展/收获？
- [ ] 有没有废话/水字数？
- [ ] 对话是否精炼？
- [ ] 断章是否恰当？
- [ ] **AI痕迹是否已清除？（见humanizer检查清单）**

### 第五步：提交审核

完成自检后，将章节提交给编辑审核。

## 质量要求

### 基础要求

- 每章3000-5000字
- 节奏明快，不拖沓
- 爽点清晰，情绪到位
- 断章有悬念
- **无AI写作痕迹**

### 进阶要求

- 场景画面感强
- 人物性格鲜明
- 斗法/冲突有张力
- 伏笔埋设自然
