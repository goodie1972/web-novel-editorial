# 网文写手技能包

## 技能概述

本技能包定义网文撰写环节的执行步骤和质量标准。

## 触发条件

- 收到研究简报和创作大纲
- 需要将大纲扩展为具体章节内容

## AI去味要求（必须遵守⭐）

**所有产出必须去除AI写作痕迹**，详见 [humanizer.md](../../references/humanizer.md)

核心原则：
- 删除填充短语和开场白（"值得注意的是"、"需要强调的是"）
- 打破公式结构（避免"不仅...而且..."、三段式列举）
- 变化节奏（长短句混合，段落结尾多样化）
- 信任读者（不过度解释、不手把手引导）
- 删除金句式表达（听起来像名言警句的句子）

写作时：
- 避免使用AI高频词：此外、与此同时、至关重要、深入探讨、持久的、显著的
- 使用简单的"是/有"结构，避免"作为/充当/拥有"
- 大声朗读检查是否自然

## 工作流程

### 第一步：确认大纲

收到编辑审核通过的大纲后，写手需：

1. 仔细阅读大纲，理解整体结构和爽点布局
2. 确认每一章的核心爽点
3. 确认人物设定和世界观细节
4. 如有疑问，先与编辑沟通

### 第二步：世界观构建

根据大纲需求，完善世界观设定：

1. 修炼体系（如果需要）
2. 势力分布
3. 货币/资源体系
4. 重要地名/场所

### 第三步：章节撰写

按照以下模板结构撰写每一章：

**章开篇**：
- 承接上文钩子（200-300字）
- 本章背景引入

**章中段**：
- 情节铺垫（500-1000字）
- 冲突种子埋入

**章高潮**：
- 核心冲突爆发（1000-2000字）
- 爽点释放

**章结尾**：
- 胜负已分或悬念留下
- 甩出下一章钩子

### 第四步：自检

完成每章后，自检以下内容：

- [ ] 本章爽点是否明确？
- [ ] 是否有钩子吸引读者往下看？
- [ ] 主角是否有进展/收获？
- [ ] 有没有废话/水字数？
- [ ] 对话是否精炼？
- [ ] 断章是否恰当？
- [ ] **AI痕迹是否已清除？（见humanizer检查清单）**

### 第五步：提交审核

完成自检后，将章节提交给编辑审核。

## 质量要求

### 基础要求

- 每章3000-5000字
- 节奏明快，不拖沓
- 爽点清晰，情绪到位
- 断章有悬念
- **无AI写作痕迹**

### 进阶要求

- 场景画面感强
- 人物性格鲜明
- 斗法/冲突有张力
- 伏笔埋设自然

- 章节内容写入：`outputs/chapters/chapter-XX.md`
- 设定文档写入：`outputs/world/`
- 大纲变更需先与编辑沟通
