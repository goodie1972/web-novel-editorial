# 网文写手技能包

## 技能概述

本技能包定义网文撰写环节的执行步骤和质量标准。

## 触发条件

- 收到研究简报和创作大纲
- 需要将大纲扩展为具体章节内容

## 记忆库使用规则（必须遵守⭐）

**全员讨论会后，所有资料必须通过记忆库获取。写作前必须查询记忆库。**

### 写作前查询（必须）

```
开始写作 → 从记忆库查询设定 → 基于查询结果写作 → 完成后将摘要写入记忆库
```

查询内容：
| 查询项 | 集合 | 用途 |
|--------|------|------|
| 人物设定 | characters | 确保人物性格、外貌、行为一致 |
| 世界观 | world | 确保设定细节准确 |
| 技能设定 | skills | 确保技能效果、威力一致 |
| 最近章节 | chapters | 确保情节衔接 |
| 伏笔 | foreshadowing | 确认本章需要埋入/回收的伏笔 |

详细配置见 [chroma-integration.md](../../references/chroma-integration.md)

## AI去味要求（必须遵守⭐）

**所有产出必须去除AI写作痕迹，这是写手铁律。** 详见 [humanizer.md](../../references/humanizer.md)

### 五条核心规则

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 去除开场白、强调性拐杖词（"值得注意的是""需要强调的是"） |
| 打破公式结构 | 避免二元对比、三段式列举、修辞性设置 |
| 变化节奏 | 长短句混合，段落结尾多样化 |
| 信任读者 | 直接陈述事实，不过度解释 |
| 删除金句 | 不写可引用式的"名言警句" |

### 高频问题词汇

**禁止使用**：此外、与此同时、至关重要、深入探讨、持久的、显著的、作为...代表

**禁止结构**：不仅...而且...、从X到Y、这不仅仅是...而是...

### 产出后自检

每次提交前检查：
- [ ] 本章爽点是否明确？
- [ ] 是否有钩子吸引读者往下看？
- [ ] 主角是否有进展/收获？
- [ ] 有没有废话/水字数？
- [ ] 对话是否精炼？
- [ ] 断章是否恰当？
- [ ] **AI痕迹初步标记**（标记可疑处，提交给编辑判断）

## 工作流程

### 第一步：确认大纲

收到编辑审核通过的大纲后，写手需：

1. 仔细阅读大纲，理解整体结构和爽点布局
2. 确认每一章的核心爽点
3. 确认人物设定和世界观细节
4. 如有疑问，先与编辑沟通

### 第二步：世界观构建

根据大纲需求，完善世界观设定：

1. 修炼体系（如果需要）
2. 势力分布
3. 货币/资源体系
4. 重要地名/场所

### 第三步：章节撰写

**核心理念**：不要试图一次性生成整章。采用分步生成模式。

#### 分步生成模式（推荐⭐）

**步骤一：先写场景一**
> 写场景一：[场景名+地点+时间]。约300字。
> 要求：xxx

生成后，人工检查/修改。

**步骤二：基于修改后的内容，写场景二**
> 基于上面的内容，写场景二：[场景名]。约500字。
> 要求：xxx

继续场景三、场景四...

#### 场景拆分原则

根据章节细纲中的场景拆分表，每个场景单独生成：
- 场景1：铺垫
- 场景2：发展
- 场景3：高潮
- 场景4-5：结尾+钩子

#### 专项指令优化

当某个部分描写不足时，使用专项指令：

| 问题 | 优化指令 |
|------|----------|
| 描写不足 | "这段打斗太干瘪了。请重写，加入动作分解、周围环境破坏的描写、以及主角呼吸急促的细节，字数扩充到800字。" |
| 对话太假 | "这段对话太像说明书了。请重写，让老王头说话带点方言味，多停顿，多些潜台词，不要直接把信息全说出来。" |
| 情感不到位 | "主角此刻应该是愤怒但隐忍的。请通过他握紧扁担的手指发白、眼神的细微变化来表现，不要直接写'他很生气'。" |

#### 按模板结构撰写

整体章结构：

**章开篇**：
- 承接上文钩子（200-300字）
- 本章背景引入

**章中段**：
- 情节铺垫（500-1000字）
- 冲突种子埋入

**章高潮**：
- 核心冲突爆发（1000-2000字）
- 爽点释放

**章结尾**：
- 胜负已分或悬念留下
- 甩出下一章钩子

### 第四步：自检

完成每章后，自检以下内容：

- [ ] 本章爽点是否明确？
- [ ] 是否有钩子吸引读者往下看？
- [ ] 主角是否有进展/收获？
- [ ] 有没有废话/水字数？
- [ ] 对话是否精炼？
- [ ] 断章是否恰当？
- [ ] **AI痕迹是否已清除？（见humanizer检查清单）**

### 第五步：提交审核

完成自检后，将章节提交给编辑审核。

## 质量要求

### 基础要求

- 每章3000-5000字
- 节奏明快，不拖沓
- 爽点清晰，情绪到位
- 断章有悬念
- **无AI写作痕迹**

### 进阶要求

- 场景画面感强
- 人物性格鲜明
- 斗法/冲突有张力
- 伏笔埋设自然
