# 网文编辑技能包

## 技能概述

本技能包定义网文审核环节的执行步骤和判断标准。

## 触发条件

- 收到写手提交的研究简报
- 收到写手提交的大纲
- 收到写手提交的章节初稿

## 记忆库使用规则（必须遵守⭐）

**全员讨论会后，所有资料必须通过记忆库获取。审核时必须查询记忆库进行对比。**

### 审核前查询（必须）

```
开始审核 → 从记忆库查询设定 → 与稿件对比 → 标注不一致处 → 给出审核意见
```

查询内容：
| 查询项 | 集合 | 用途 |
|--------|------|------|
| 人物设定 | characters | 对比人物描写是否一致 |
| 世界观 | world | 对比设定是否准确 |
| 战力体系 | world | 对比战力是否合理 |
| 历史章节 | chapters | 对比情节是否衔接 |
| 伏笔状态 | foreshadowing | 确认伏笔埋入/回收是否正确 |

详细配置见 [chroma-integration.md](../../references/chroma-integration.md)

## AI去味审核（必须⭐）

**所有产出必须通过AI痕迹检测，这是编辑铁律。** 详见 [humanizer.md](../../references/humanizer.md)

### 五条核心规则

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 去除开场白、强调性拐杖词（"值得注意的是""需要强调的是"） |
| 打破公式结构 | 避免二元对比、三段式列举、修辞性设置 |
| 变化节奏 | 长短句混合，段落结尾多样化 |
| 信任读者 | 直接陈述事实，不过度解释 |
| 删除金句 | 不写可引用式的"名言警句" |

### 高频问题词汇

**禁止使用**：此外、与此同时、至关重要、深入探讨、持久的、显著的、作为...代表

**禁止结构**：不仅...而且...、从X到Y、这不仅仅是...而是...

### AI痕迹检测要点

| 检测项 | 问题表现 | 判定 |
|--------|----------|------|
| 填充短语 | "值得注意的是"、"需要强调的是"、"此外" | 删 |
| 公式结构 | "不仅...而且..."、三段式列举 | 改 |
| AI高频词 | "至关重要""深入探讨""持久的""显著的" | 换 |
| 系动词回避 | "作为...代表"而非"是" | 改 |
| 破折号滥用 | 连续使用破折号制造"有力"感 | 改 |
| 过度解释 | 手把手引导读者 | 删 |
| 金句式表达 | 读起来像名言警句 | 改 |
| 语气词 | "当然！""希望对您有帮助！" | 删 |

## 审核类型

### 1. 大纲审核

审核写手提交的大纲设计：

- 题材选择是否合理
- 爽点布局是否密集
- 结构是否清晰递进
- 世界观是否自洽
- 是否有差异化亮点

### 2. 初稿审核

审核章节内容：

- 节奏是否明快
- 爽点是否到位
- 断章是否有悬念
- 人物是否立体
- 文笔是否流畅
- 有无明显错误
- **是否有AI痕迹**

### 3. 终审

最终交付前的全面审核：

- 全文逻辑是否通顺
- 伏笔是否都回收
- 战力是否崩塌
- 高潮是否足够爽
- 结局是否满意

## 审核流程

### 第一步：阅读理解

完整阅读需要审核的内容，不要跳读。

### 第二步：对照清单

使用 `checklist.md` 中的验收清单逐项检查。

### 第三步：AI痕迹检测

使用humanizer.md中的检查清单逐项排查AI写作痕迹。

### 第四步：问题定位

如果发现问题，需要：

1. 标注具体位置（章节/段落）
2. 说明违反了哪条标准
3. 给出修改建议

### 第五步：判定结果

| 判定 | 含义 | 后续动作 |
|------|------|----------|
| **通过** | 审核通过 | 进入下一环节 |
| **不通过** | 需要修改 | 发起讨论 |

## 判定标准

### ⭐ 关键项（必须通过）

- 有明确的爽点
- 有悬念钩子
- 主角有进展
- 无原则性错误
- **无AI写作痕迹**

### 一般项（允许少量不通过）

- 断章技巧
- 文笔精炼度
- 场景渲染
- 人物细节

## 讨论协议

编辑打回不是简单地说"不通过"，必须：

1. **引用具体位置**："第3章第5段..."
2. **说明违反标准**："这里违反了爽点设计原则..."或"这里有AI写作痕迹..."
3. **给出修改方向**："建议增加主角的反杀情节..."或"建议删除填充短语..."

写手可以：
- 接受修改建议
- 反驳（但必须给出理由或替代方案）

如无法达成共识，总编裁决。

## 审核频率

- 大纲：一次性审核
- 初稿：每10章审核一次
- 终稿：全文审核

## 文件操作规则

- 审核意见写入：`outputs/reviews/review-XX.md`
- 通过后标记状态更新
