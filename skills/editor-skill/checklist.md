# 网文审核清单

本文档定义网文各环节的验收标准。

**重要**：每次审核必须逐项勾选，缺一不可！

---

## 一、AI痕迹检测（必须⭐）

使用 [humanizer.md](../../references/humanizer.md) 进行检测

| 检查项 | 问题表现 | 状态 |
|--------|----------|------|
| 填充短语 | "值得注意的是"、"需要强调的是"、"此外" | [ ] |
| 公式结构 | "不仅...而且..."、三段式列举 | [ ] |
| AI高频词 | "至关重要""深入探讨""持久的""显著的" | [ ] |
| 系动词回避 | "作为...代表"而非简单"是" | [ ] |
| 破折号滥用 | 连续使用破折号 | [ ] |
| 过度解释 | 手把手引导读者 | [ ] |
| 金句式表达 | 读起来像名言警句 | [ ] |
| 语气词 | "当然！""希望对您有帮助！" | [ ] |
| 弯引号 | ""而非"" | [ ] |
| 虚假归因 | "专家表示""观察者指出"无具体来源 | [ ] |

### 快速检查清单

- [ ] 连续三个句子长度相同？ → 打断其中一个
- [ ] 段落以简洁的单行结尾？ → 变换结尾方式
- [ ] 解释隐喻或比喻？ → 相信读者能理解
- [ ] 使用了"此外""然而"等连接词？ → 考虑删除
- [ ] 三段式列举？ → 改为两项或四项

## 三、初稿审核清单（黄金三章）

### 3.1 黄金三章 ⭐

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 第一章 | 人设立住+金手指展示+冲突引入 | 必须 |
| 第二章 | 冲突升级+世界观展示+目标明确 | 必须 |
| 第三章 | 首次小胜+长期目标暗示+钩子 | 必须 |

### 3.2 爽点质量 ⭐

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 每章爽点 | 至少1个明确爽点 | 必须 |
| 爽点力度 | 情绪到位，不过于平淡 | 必须 |
| 叠加效果 | 有场景渲染和旁观者视角 | 建议 |

### 3.3 节奏把控

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 节奏明快 | 无明显拖沓 | 必须 |
| 章节完整 | 有铺垫-高潮-收尾结构 | 必须 |
| 张力保持 | 无冷场段落 | 必须 |

### 3.4 断章技巧

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 悬念设置 | 每章有钩子 | 必须 |
| 断章位置 | 在高潮点或转折点断开 | 必须 |
| 钩子类型 | 悬念/危机/反转/行动 | 建议多元化 |

### 3.5 人物塑造

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 主角代入感 | 行为合理，有魅力 | 必须 |
| 对话精炼 | 每句不超过20字 | 建议 |
| 人物性格 | 前后一致，有记忆点 | 必须 |

### 3.6 文笔质量

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 画面感 | 场景清晰可想象 | 建议 |
| 动作描写 | 为主，心理描写为辅 | 必须 |
| 错别字 | 无明显错误 | 必须 |

### 3.7 雷区检测

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 主角憋屈 | 连续不超过3章 | 必须 |

## 五、判定规则

### PASS 条件

- 所有 ⭐ 必须项全部通过
- **AI痕迹检测不超过2项**
- 一般项允许1-2项不通过，但需标注"建议修改"

### FAIL 条件

- 任何 ⭐ 必须项不通过 → 直接 FAIL
- **AI痕迹检测超过2项 → FAIL**
- 3项以上一般项不通过 → FAIL
- 存在原则性错误（主角人设崩塌、战力完全混乱）→ 直接 FAIL

### 讨论触发

- FAIL 后必须进入讨论环节
- 编辑需给出具体位置和修改建议
- 写手可接受或反驳
- 无法达成共识时总编裁决
