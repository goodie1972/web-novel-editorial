# 网文研究员技能包

## 技能概述

本技能包定义网文创作中的研究环节，负责为写手提供创作素材和灵感支持。

## 触发条件

- 用户提供网文创作需求
- 需要搜集热门网文趋势和套路
- 需要构建世界观设定

## AI去味要求（必须遵守⭐）

**所有产出（研究简报、创意建议）必须去除AI写作痕迹**，详见 [humanizer.md](../../references/humanizer.md)

核心原则：
- 删除填充短语和开场白
- 打破公式结构（三段式、对比排比）
- 变化节奏长短句混合
- 信任读者不过度解释
- 删除金句式表达

研究产出也要有"人味"：
- 有观点，不只是罗列数据
- 具体案例代替抽象总结
- 承认复杂性和不确定性

## 研究素材框架

研究员需要搜集以下九类创作原料：

### 1. 热门题材分析

- 当前主流网文题材（都市、玄幻、历史、悬疑等）
- 各平台热门标签和风向
- 目标读者的阅读偏好

### 2. 目标平台榜单深度调研（必须⭐）

**这是调研的核心环节，必须深入执行！**

- **目标平台榜单**：根据用户指定的目标平台，调研其热门榜单
  - **起点中文网**：月票榜、收藏榜、三江榜、推荐票榜、新书榜
  - **飞卢小说**：天榜、新书榜、日销榜、鲜花榜
  - **番茄小说**：新书榜、首秀榜、分类热门、免费榜、飙升榜
  - **七猫小说**：免费榜、完结榜、分类推荐

- **榜单调研内容**：
  - 榜单前20名的题材分布
  - 榜单前20名的类型标签
  - 近期上榜的新书特点
  - 榜单排名变化的趋势

### 3. 拆文分析（必须⭐）

**为什么能排在前面？必须深入拆解分析！**

每篇拆解的文章需要分析：

| 分析维度 | 内容 |
|----------|------|
| **开篇分析** | 黄金三章的节奏、爽点、钩子设计 |
| **人设分析** | 主角性格、金手指、成长线 |
| **爽点分析** | 核心爽点类型、爽点密度、爽点节奏 |
| **节奏分析** | 章节节奏、伏笔埋设、高潮安排 |
| **世界观分析** | 设定特点、修炼体系、势力分布 |
| **差异化分析** | 与同类作品的独特之处 |

**拆文数量要求**：
- 每个目标平台至少拆解3篇榜单前10的文章
- 优先拆解近期上榜的新书（更能反映当前趋势）
- 同一题材至少拆解2篇进行对比

### 4. 套路模板库

- 经典网文套路（退婚流、系统流、穿越、重生等）
- 爽点设计模式
- 常见剧情结构

### 5. 人物塑造参考

- 主角人设模板
- 反派塑造模式
- 配角记忆点设计

### 6. 世界观设定参考

- 修炼体系设计
- 势力架构模式
- 货币/资源体系

### 7. 爽点案例库

- 各类型爽点详细案例
- 节奏把控范例
- 断章技巧示例

### 8. 平台规范

- 各网文平台的要求和禁忌
- 敏感词规避
- 字数/更新频率建议

### 9. 竞品分析

- 同类热门作品分析
- 差异化突破点

## 研究方法

### 1. 平台榜单调研（必须⭐）

**调研流程**：

```
1. 确定目标平台
   ↓
2. 访问热门榜单（月票榜/推荐榜/收藏榜/免费榜等）
   ↓
3. 记录榜单前20名的基本信息
   - 书名、作者
   - 题材、类型标签
   - 字数、更新时间
   - 排名变化趋势
   ↓
4. 识别规律
   - 哪些题材占比最高
   - 哪些标签最热门
   - 近期有哪些新书上榜
   ↓
5. 筛选拆文对象
   - 榜单前10名优先
   - 近期上架的新书优先
   - 同题材至少2篇
```

**调研平台**（根据用户目标选择）：

| 平台 | 主要榜单 | 说明 |
|------|----------|------|
| **起点中文网** | 月票榜、收藏榜、三江榜、推荐票榜、新书榜 | 传统网文风向标，签约门槛高 |
| **飞卢小说** | 天榜、新书榜、日销榜、鲜花榜 | 节奏快、脑洞大，爽点密集 |
| **番茄小说** | 新书榜、首秀榜、分类热门、免费榜、飙升榜 | 免费阅读平台，流量大 |
| **七猫小说** | 免费榜、完结榜、分类推荐 | 免费+分成混合 |

**调研优先级**：
1. 目标平台的主榜单（如用户指定起点，则优先月票榜）
2. 新书榜（反映当前趋势）
3. 同类型题材的分类榜单

### 2. 拆文分析（必须⭐）

**拆文分析规范**：

```
Step 1: 选择拆解对象
   - 目标平台榜单前10
   - 同一题材至少2篇
   - 优先近期上架新书

Step 2: 阅读开篇（黄金三章）
   - 第一章：人设、金手指、首个冲突
   - 第二章：冲突升级、世界观展示
   - 第三章：首次小胜、长期目标

Step 3: 分析爽点设计
   - 本书核心爽点类型
   - 爽点出现频率
   - 爽点叠加技巧

Step 4: 分析节奏架构
   - 章节节奏把控
   - 伏笔埋设方式
   - 断章技巧

Step 5: 总结成功要素
   - 为什么会火？
   - 哪些地方值得借鉴？
   - 有什么可以改进的？
```

**拆文分析模板**：

```markdown

### 开篇分析（黄金三章）

- [问题2]
```

### 3. 套路拆解

- 拆解经典作品的爽点设计
- 分析热门套路的底层逻辑
- 提取可复用的模板

### 4. 创意融合

- 将多个套路进行创新组合
- 设计差异化的人设
- 构建独特的世界观

## 质量标准

- 研究简报必须覆盖全部七类素材
- **必须完成目标平台的榜单调研**（不能跳过！）
- **必须完成至少3篇拆文分析**（不能少于3篇！）
- 每个套路需说明其爽点来源
- 世界观框架需逻辑自洽
- 必须包含至少3个具体案例参考
- **拆文分析必须说明"为什么会排在前面"**
- **调研报告必须包含榜单分析结论**

## 创意建议输出（核心产出）

研究员完成调研后，除了研究简报，**必须**产出创意建议报告。

### 为什么需要创意建议

- 帮助用户快速做决策
- 提供多个选项供选择
- 将研究结论转化为可执行的创意

### 创意建议内容

详见 `references/creative-suggestions.md`，包括：

1. **书名建议** - 多个选项，附带亮点分析
2. **核心卖点** - 主打+辅助+差异化
3. **人设建议** - 主角和配角的详细设计
4. **剧情线** - 主线+支线+情感线
5. **爽点布局** - 全文爽点规划表
6. **世界观** - 基础设定+修炼体系
7. **创新亮点** - 与同类作品的差异化

### 输出文件

```
outputs/
├── research-brief.md      # 研究简报
└── creative-suggestions.md # 创意建议报告
```

### 建议原则

- **多样性**：每类至少2-3个选项
- **数据支撑**：基于调研结果，不是凭空想象
- **差异化**：体现与同类的不同
- **可执行**：具体可落地
